# Logi Voyage 多平台发布路线

## 结论

当前项目最适合做成“内容导出与发布辅助台”，而不是直接做全自动多平台发布器。

推荐路线：

1. `scripts/data.js` 继续作为站内文章源。
2. 增加统一导出层，把同一篇文章转换为 Markdown、公众号 HTML、知乎 HTML、小红书文案和图片卡片脚本。
3. 公众号优先走草稿箱 API，知乎优先走 HTML 预览与手动粘贴，小红书优先走创作服务平台人工确认。
4. 所有平台的最终发布动作都保留人工确认，避免账号风控、内容审核、登录态失效和平台规则变化造成不可控风险。

## 平台路线

### 站内博客

现状：

- `editor.html` 生成文章 JSON。
- 手动粘贴到 `scripts/data.js` 的 `window.BLOG_POSTS`。
- `index.html`、`posts.html`、`post.html` 自动读取并渲染。

优化：

- 给编辑器增加“导出”区域：
  - 导出 `BLOG_POSTS` JSON。
  - 导出 Markdown。
  - 导出公众号 HTML。
  - 导出知乎 HTML。
  - 导出小红书文案。
- 不改变现有静态架构，不引入构建工具。

### 微信公众号

推荐自动化等级：高，但只到草稿箱。

路线：

1. 从文章对象生成 Markdown 或 HTML。
2. 使用 `wechat-article-publisher` 写入公众号草稿箱。
3. 用户进入公众号后台预览、改封面、检查图片和样式。
4. 用户手动发布。

注意：

- 标题控制在平台限制内。
- 摘要建议从 `summary` 字段生成。
- 封面优先使用 `cover` 字段。
- 站内图片如果是本地路径，需要先转为可访问 URL 或由发布服务上传。

### 知乎

推荐自动化等级：中，优先半自动。

路线：

1. 从文章对象生成 Markdown。
2. 用 `md-to-zhihu` 转换成知乎兼容 HTML。
3. 生成一个预览页，用户复制内容到知乎编辑器。
4. 自动发布脚本仅作为可选项，默认不启用。

注意：

- 技术复盘、工具教程、项目踩坑类文章适合知乎。
- 不建议把公众号排版原样搬到知乎，知乎更适合结构清晰、解释充分的长文。
- 话题标签可以从 `category` 和标题关键词生成。

### 小红书

推荐自动化等级：低到中，优先做内容包装，不直接绕过官方入口。

可用官方入口：

- 小红书创作服务平台：适合创作者上传视频、查看数据、管理内容。入口：`https://creator.xiaohongshu.com/`
- 小红书开放平台：当前公开文档重点更偏电商、订单、商品、素材、商家自研系统等能力。入口：`https://open.xiaohongshu.com/`
- 蒲公英平台：偏商业合作、达人/品牌合作，不是普通个人博客的一键发文 API。入口：`https://pgy.xiaohongshu.com/`

建议路线：

1. 从长文生成小红书版本：
   - 标题：更短、更口语化。
   - 正文：拆成 6 到 10 个信息块。
   - 标签：生成 5 到 8 个话题。
   - 引导：保留“主页/原文/评论区见”的人工可编辑文本。
2. 生成图文笔记素材：
   - 封面图。
   - 3 到 9 张卡片图。
   - 每张卡片承载一个观点、步骤或截图。
3. 输出一个 `xhs.md` 或 `xhs.json`：
   - `title`
   - `caption`
   - `hashtags`
   - `images`
   - `source_post_id`
4. 用户在小红书创作服务平台或 App 内手动发布。

暂不建议：

- 抓包逆向小红书创作平台接口。
- 保存或复用个人登录 Cookie 做无人值守发布。
- 使用不明第三方代发 API 直接接管账号发布。

可谨慎探索：

- 二维码式发布服务：系统提交标题、文案、图片，生成二维码，用户手机扫码后在小红书端确认发布。
- 前提是服务方信誉、数据处理、账号授权方式、价格和隐私风险都可接受。

## 推荐信息架构

```text
BLOG_POSTS article
  -> platform-exporter
    -> site-json
    -> markdown
    -> wechat-html
    -> zhihu-html
    -> xhs-caption
    -> xhs-card-plan
```

## 第一版实现范围

优先做这三件事：

1. 在编辑器里增加“多平台导出”按钮。
2. 新增纯前端导出逻辑，不引入 package.json。
3. 输出公众号、知乎、小红书三个平台的可复制内容。

第一版不做：

- 不直接登录任何平台。
- 不存平台 Cookie。
- 不接不明第三方发文 API。
- 不做批量发布。

## 后续增强

- 增加 `exports/` 目录，把每次导出保存为文件。
- 增加封面/卡片图片生成脚本。
- 增加“平台适配评分”：标题长度、摘要长度、图片缺失、标签数量、敏感词提示。
- 公众号草稿箱发布作为高级按钮，仅在配置 `WECHAT_API_KEY` 后启用。
- 知乎自动草稿作为实验功能，仅在用户明确授权后启用。
