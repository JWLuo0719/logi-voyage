# 《Web前端开发》课程设计视频讲解稿

## 1. 开场介绍

大家好，我的课程设计题目是《个人博客网站开发》，项目名称是 `Logi Voyage`。

这个项目是一个基于 `HTML、CSS 和 JavaScript` 实现的多页面个人博客网站，主要用于展示我的前端学习记录、AI 工具实战内容，以及个人资料和账号入口。

本次视频我会从五个部分进行讲解：

1. 课题目标和整体设计思路
2. 网站主要页面展示
3. 核心功能实现说明
4. 部分关键代码讲解
5. 项目总结

---

## 2. 项目目标与设计思路

这个项目对应课程设计题目中的“个人博客网站开发”。

老师要求至少完成以下几点：

1. 使用 HTML、CSS 和 JavaScript 实现博客网站基本结构
2. 至少包含首页、文章列表页、文章详情页和个人资料页
3. 实现响应式设计，保证不同设备下都能正常显示
4. 多页面、风格统一、元素丰富、界面美观

我的设计思路是：

先确定网站的信息结构，再统一视觉风格，最后补充交互功能。

在信息结构上，我把网站分成四个核心页面：

1. 首页：作为整个网站的入口
2. 文章列表页：集中展示所有文章
3. 文章详情页：展示具体文章内容
4. 个人资料页：展示作者信息和账号入口

在视觉风格上，我采用了统一的浅色卡片风格，并通过 CSS 变量统一控制颜色、圆角、阴影和间距。

在交互上，我主要实现了：

1. 首页推荐文章动态渲染
2. 文章列表页搜索和分类筛选
3. 文章详情页根据 URL 参数动态切换内容
4. 移动端导航折叠与展开

---

## 3. 网站页面展示

### 3.1 首页展示

现在先看首页。

首页主要包含以下几个模块：

1. 顶部导航栏
2. 首页主视觉区域
3. 个人头像和账号入口
4. 网站概览介绍
5. 最近更新文章
6. 站点说明和内容方向

首页的作用主要是让用户快速知道这个网站是做什么的，以及从哪里进入其它页面。

在布局上，首页采用卡片化设计，整体结构清晰，信息层级比较明确。

---

### 3.2 文章列表页展示

接下来是文章列表页。

文章列表页的主要功能是集中展示所有文章，并支持用户快速筛选内容。

页面中包括：

1. 页面横幅介绍
2. 搜索框
3. 分类筛选按钮
4. 文章卡片列表

用户可以输入关键词进行搜索，也可以点击分类按钮筛选不同类型的文章，比如 AI 工具、学习记录、开发实践等。

这个页面体现了 JavaScript 对页面内容的动态更新能力。

---

### 3.3 文章详情页展示

接下来展示文章详情页。

这个页面的主要作用是显示某一篇文章的完整内容。

详情页包括：

1. 文章标题
2. 分类、日期、阅读时长等信息
3. 封面图
4. 正文内容
5. 推荐阅读
6. 作者补充信息

这个页面不是为每篇文章单独写一个 HTML，而是通过一个通用模板页面 `post.html`，根据地址栏中的 `id` 参数动态切换内容。

这样做的好处是：

1. 结构更统一
2. 扩展更多文章更方便
3. 减少重复代码

---

### 3.4 个人资料页展示

最后是个人资料页。

这个页面主要展示作者相关信息，包括：

1. 头像和简介
2. 技能方向
3. 常用工具
4. 账号入口
5. 成长时间线

这个页面和首页形成互补：首页偏入口和概览，资料页偏完整介绍。

## 4. 核心功能说明

### 4.1 多页面结构

本项目已经完成了课程要求中的四个核心页面：

1. `index.html`
2. `posts.html`
3. `post.html`
4. `profile.html`

所以在“多页面结构”这一项上是达标的。

---

### 4.2 响应式设计

在响应式方面，我主要通过 CSS 媒体查询实现不同设备适配。

例如：

1. 在桌面端使用多列布局
2. 在平板和手机端切换成单列布局
3. 小屏幕下导航栏会折叠成按钮
4. 卡片、文字和图片尺寸也会相应调整

这样可以保证网站在不同设备上都能良好显示。

---

### 4.3 数据统一管理

为了避免在多个页面重复写文章内容，我把文章数据统一写在 `scripts/data.js` 中。

每篇文章的数据包括：

1. id
2. title
3. category
4. date
5. readTime
6. summary
7. cover
8. content

这样首页、列表页和详情页都可以共用同一份数据，维护起来更方便。

---

### 4.4 交互功能

本项目实现了以下主要交互：

1. 首页自动展示最新三篇文章
2. 列表页按关键词搜索文章
3. 列表页按分类筛选文章
4. 详情页根据参数动态切换内容
5. 移动端导航按钮展开和收起

这些功能虽然不复杂，但能够体现 JavaScript 在前端项目中的实际应用。

## 5. 关键代码讲解

下面简单讲解几个核心代码部分。

### 5.1 首页推荐文章渲染

首页推荐文章的逻辑写在 `scripts/home.js` 中。

核心思路是：

先获取首页文章容器，然后从公共数据 `BLOG_POSTS` 中取出前三篇文章，再通过统一的卡片生成函数渲染到页面中。

讲解时可以这样说：

“这里我没有手写三张文章卡片，而是直接读取数据数组中的前三条内容，调用统一的 `createPostCard` 方法生成 HTML，这样首页和文章列表页可以复用同一套卡片结构。”

---

### 5.2 文章列表页搜索与筛选

文章列表页逻辑写在 `scripts/posts.js` 中。

这里主要做了两件事：

1. 根据数据自动生成分类按钮
2. 根据分类和关键词共同过滤文章

讲解时可以这样说：

“列表页的筛选不是写死的，而是从文章数据中自动提取分类，再绑定点击事件。用户输入关键词或者切换分类时，都会重新执行 `render` 函数，从而更新页面中的文章列表。”

---

### 5.3 文章详情页动态切换

详情页逻辑写在 `scripts/post.js` 中。

它的关键点在于读取 URL 参数：

1. 先通过 `URLSearchParams` 获取地址栏中的 `id`
2. 再在文章数据数组中查找对应文章
3. 找到后把标题、摘要、正文等内容渲染出来

讲解时可以这样说：

“这样做以后，不需要给每篇文章单独建立一个 HTML 页面，只需要一个通用模板页就可以显示所有文章内容，扩展性会更好。”

---

### 5.4 移动端导航交互

移动端导航逻辑写在 `scripts/site.js` 中。

核心做法是：

1. 点击按钮时给导航栏切换 `is-open` 类名
2. 同时更新 `aria-expanded` 属性

讲解时可以这样说：

“这个功能主要是为了解决手机端导航不方便直接平铺显示的问题。通过 class 切换和 CSS 配合，可以让导航菜单在小屏幕下折叠起来，点击按钮后再展开。”

## 7. 遇到的问题与解决方法

在制作过程中，我主要遇到了几个问题：

### 问题一：文章内容重复维护

如果每个页面都分别写文章内容，后期修改很麻烦。

解决方法是把文章数据统一放到 `data.js` 中，让多个页面共享数据。

### 问题二：文章详情页数量会越来越多

如果每篇文章都单独建立一个详情页，结构会很冗余。

解决方法是使用 `post.html + id 参数` 的方式动态切换内容。

### 问题三：移动端布局容易拥挤

桌面端多列布局放到手机上后会显得很挤。

解决方法是使用媒体查询，在小屏幕下改成单列布局，并增加折叠导航。
